# CrushTask PRD（产品需求文档）

---

## 1. 产品定位

CrushTask 是基于现有代码大模型（OpenCode、Claude、GPT 等）构建的 AI 驱动任务执行编排系统，不替代底层 AI 能力，而是在 AI 代码生成能力之上构建完整任务执行闭环，实现「需求输入 → 自动执行 → 可运行结果输出」的全流程自动化。

- **左侧**：产品功能任务树（状态视图+产品主表双维度）
- **中部**：单个任务的全生命周期控制台（任务定义 → 上下文 → AI执行 → 验收）
- **核心理念**：以任务为核心驱动，AI 完成代码生成、编译、构建、运行等全流程执行，人类聚焦审核验收与需求定义
- **目标用户**：开发者、产品经理、技术团队

---

## 2. 核心功能

### 2.1 左侧任务树

**布局结构**：
- 纯树形结构，无标签页

**展示顺序**：
1. 🚀 进行中（状态视图，筛选in_progress任务）
2. 📦 CrushTask（产品主表，唯一数据存储）

**约束规则**：
- 仅显示进行中状态视图
- 暂不支持已完成视图
- 不支持自定义视图
- 产品支持后续扩展多产品
- 数据统一存储于产品主表，状态视图为动态筛选结果

### 2.2 中部任务详情

**四个标签页**：
1. 任务本体
2. 上下文
3. AI执行
4. 验收

**任务本体（模块化结构）**：
- 任务目标：可编辑文本
- 输入/输出：结构化展示
- 约束条件：列表式
- 验收标准：可勾选清单

**上下文**：
- 关联工程
- 技术栈
- 相关文件
- 依赖任务

**AI执行**：
- AI对话面板（底部固定输入框，遵循主流AI插件交互范式）
- 执行日志（含代码生成、编译、构建、运行全流程记录）
- 代码Diff对比
- 任务执行结果预览（展示可运行的最终交付物，非代码/文件）

**验收**：
- 验收清单（与任务本体同步）
- 通过/驳回按钮

### 2.3 AI执行引擎

**输入规范**：
- 仅读取结构化任务字段（goal/input/output/constraints/acceptanceCriteria）
- 不处理自由文本

**输出类型**：
- 代码生成
- 编译构建日志
- 可运行程序/交付物
- 执行结果返回

## 3. UI设计规范

### 3.1 布局规格

- 左侧宽度：固定260px
- 中央区域：自适应
- 标签页：固定顶部
- 中央区域上半部分：多标签页+任务执行结果预览区
- 中央区域下半部分：固定AI对话输入框

### 3.2 视觉风格

- 主色：#165DFF（蓝色）
- 字体：无衬线字体（系统默认）
- 模块间距：12px
- 使用分隔线区分模块
- 禁止大段连续文本
- 预览区突出展示可运行结果，弱化代码细节

---

## 4. 交互规则

### 4.1 左侧任务树

- 点击原子任务 → 刷新中央详情
- 点击模块/子功能 → 折叠/展开
- 双击 → 无额外行为

### 4.2 中部任务详情

- 模块支持折叠/展开
- 所有字段可编辑
- 验收清单可勾选
- 预览区支持结果刷新、全屏查看

---

## 5. 开发流程

所有功能实现与重构严格遵循「先更新PRD → 再生成UI → 再开发实现 → 再验收确认」的闭环流程，以CrushTask PRD为唯一基准，确保功能与文档完全对齐。

### 5.1 标准开发流程（6步闭环）

1. **需求确认**：明确迭代模块与范围，更新PRD对应功能描述、UI规范、验收标准
2. **UI设计**：基于更新后的PRD生成UI结构，确保左侧任务树、中部任务详情（含预览区+底部输入框）符合规范
3. **AI开发**：AI依据PRD与UI结构完成代码生成、功能实现、交互开发，实现任务全流程执行闭环
4. **功能验收**：对照PRD验收标准逐项校验，通过则标记完成，不通过则反馈修改
5. **版本迭代**：单次迭代聚焦单一模块/任务，迭代后更新PRD版本号
6. **文档同步**：所有开发修改同步至PRD，保持文档与产品一致

### 5.2 左侧任务树重构规则

**图标定义**：
- 项目节点：📦
- 模块节点：🧩
- 子功能节点：🔧
- 原子任务（未完成）：📝
- 原子任务（已完成）：✅

**颜色规则**：
- 原子任务文字颜色：#165DFF（主色）
- 其他节点使用灰色

**交互规则**：
- 仅原子任务可点击刷新中央详情
- 模块/子功能仅支持折叠展开
- 双击无额外行为

---

## 6. 验收标准

### 6.1 功能验收

- [ ] 左侧四级结构正确显示（项目→模块→子功能→原子任务）
- [ ] 中部模块化展示符合规范（目标、输入、输出、约束、验收）
- [ ] AI可解析结构化任务字段并完成全流程执行
- [ ] 任务执行结果可在预览区展示
- [ ] 验收流程完整（清单+通过/驳回）

### 6.2 UI验收

- [ ] 符合布局规范（左侧260px，中央自适应，底部固定输入框）
- [ ] 符合风格规范（主色#165DFF，无衬线字体）
- [ ] 模块之间区分清晰
- [ ] 无文档式文本展示
- [ ] 预览区展示可运行结果而非代码/文件

---

## 7. 版本迭代规则

- 单次迭代聚焦单一模块/任务
- 迭代后更新PRD版本号
- 所有修改同步至PRD
