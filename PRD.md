# CrushTask Web MVP PRD

## 产品概述
- **产品名称**: CrushTask Web
- **类型**: AI驱动的任务管理系统（Web版）
- **核心理念**: 通过人机协作完成任务，AI执行代码/文档生成，人类审核验收
- **目标用户**: 开发者、产品经理、技术团队

## 整体架构

### 布局结构
- **左侧**: 任务树（产品功能结构）固定260px
- **中央**: 任务详情页（任务本体 + 定义 + AI执行 + 上下文 + 验收）自适应
- **右侧**: 预留（暂不实现）

### 任务层级
- 模块（Module）→ 子功能（SubFeature）→ 原子任务（Task）

---

## 数据模型

### Task 类型定义
```typescript
interface Task {
  id: string;
  title: string;                    // 任务名称
  module: string;                  // 所属模块
  subFeature: string;              // 所属子功能
  status: 'pending' | 'in_progress' | 'completed' | 'rejected';
  priority: 'low' | 'medium' | 'high';
  
  // 任务本体（核心内容）
  goal: string;                    // 任务目标
  input: string;                   // 输入
  output: string;                  // 输出
  constraints: string;             // 约束条件
  acceptanceCriteria: string[];    // 验收标准清单
  
  // 上下文信息
  context: {
    project?: string;
    techStack?: string[];
    relatedFiles?: string[];
    dependentTasks?: string[];
  };
  
  // AI对话记录
  aiMessages: AIMessage[];
  
  // 子任务
  children: Task[];
}
```

## UI/UX 设计

### 1. 左侧任务树

**结构**:
- 模块名（可展开）
  - 子功能名（可展开）
    - 原子任务

**交互**:
- 点击模块/子功能 → 展开/收起
- 点击任务 → 右侧中央加载任务详情
- 选中的任务高亮显示

**样式**:
- 深色主题
- 树形缩进
- 状态颜色标识

### 2. 中央任务详情页

**垂直分段布局**:

#### 顶部：任务头部（固定）
- 任务名称
- 所属模块 → 子功能
- 状态标签
- 优先级

#### 区域1：任务本体
- 【任务目标】
- 【输入】
- 【输出】
- 【约束】
- 【验收标准】(可勾选清单)

#### 区域2：任务上下文
- 关联工程
- 技术栈
- 相关文件
- 依赖任务

#### 区域3：AI执行区
- AI执行日志
- AI对话面板

#### 区域4：验收区
- 验收清单（从任务本体同步）
- 通过/驳回按钮
