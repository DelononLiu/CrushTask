# CrushTask Web MVP PRD (中部重构版)

## 产品定位（不变）

**CrushTask**：以 AI 为核心、以产品功能为组织方式的任务执行系统。
- 左侧：产品功能任务树
- 中部：单个任务的全生命周期控制台（定义 → 上下文 → AI执行 → 验收）

---

## 中部区域重构需求（核心变更）

### 目标
将中部从"文档式文本展示"改为"模块化属性面板"，让任务呈现为一个可操作、可编辑、可被AI理解的对象，而不是一段说明文字。

### 结构（垂直分段、模块化、可折叠）

中部分为 4 个独立模块块，从上到下依次排列：

#### 1. 任务目标模块
- 标题：任务目标
- 内容：单行/多行可编辑文本
- 样式：独立区块、可折叠

#### 2. 输入输出模块
- 标题：输入 / 输出
- 内容：结构化展示（输入：xxx；输出：xxx）
- 可编辑
- 独立区块、可折叠

#### 3. 约束条件模块
- 标题：约束条件
- 内容：列表式（• 项1 • 项2）
- 可编辑
- 独立区块、可折叠

#### 4. 验收标准模块
- 标题：验收标准
- 内容：可勾选清单（☐ 项1 ☐ 项2）
- 可编辑
- 独立区块、可折叠

### 整体风格

- 简洁、专业、深蓝主题
- 模块之间有明显分隔
- 每个模块可独立折叠/展开
- 保持现有标签页（任务本体 / 上下文 / AI执行 / 验收）不变
- 只重构"任务本体"标签页内的内容展示方式

## 实现要求

### 1. 保留完全不变
- 左侧任务树
- 顶部任务头部
- 四个标签页结构

### 2. 仅重构"任务本体"标签页

将当前文档式文本：
```
【任务目标】
xxxxx

【输入】
xxxxx

【输出】
xxxxx

【约束】
xxxxx

【验收标准】
☐ xxx
```

改为模块化对象面板：
```
┌──────────────────────────┐
│ ▼ 任务目标         [展开] │
│   [可编辑文本框]         │
├──────────────────────────┤
│ ▼ 输入/输出       [展开] │
│   输入：[可编辑]         │
│   输出：[可编辑]        │
├──────────────────────────┤
│ ▼ 约束条件       [展开] │
│   • [可编辑项]         │
│   • [可编辑项]         │
├──────────────────────────┤
│ ▼ 验收标准       [展开] │
│   ☐ [可勾选]           │
│   ☑ [可勾选]           │
└──────────────────────────┘
```

### 3. 模块特性
- 每个模块可独立折叠/展开
- 模块之间有明显分隔线
- 统一的模块标题样式
- 简洁专业的深蓝主题
